// 03 — Sécurisation du Contenu

Contenu non-téléchargeable,
visible sur le site uniquement

Chaque couche de protection est implémentée. Le contenu ne peut pas être extrait, même par un développeur.

🔒 Les 6 couches de protection

  • 🎬
    Mux — Signed URLs (expiration 2h) Les vidéos ne sont jamais servies directement. Mux génère une URL signée côté serveur qui expire après 2h. Même si quelqu'un copie le lien, il ne fonctionnera plus. Impossible de télécharger via youtube-dl ou équivalent.
  • 🚫
    Désactivation click-droit & sélection de texte CSS + JS léger désactivent le clic droit sur les vidéos et les guides, et bloquent la sélection de texte sur le contenu premium. Simple mais efficace pour 99% des utilisateurs.
  • 🔐
    Supabase Row Level Security (RLS) Chaque ressource en base est protégée par des policies RLS. Un utilisateur non-authentifié, ou sans le niveau requis, reçoit littéralement un résultat vide — même s'il inspecte les requêtes API.
  • 💧
    Watermarking dynamique (email en overlay) L'email de l'utilisateur connecté s'affiche en transparence en filigrane sur les vidéos. Ça dissuade totalement le screen-recording : si la vidéo fuite, elle est tracée. C'est ce que fait Masterclass.
  • 📄
    PDF viewers sécurisés (aucun téléchargement) Les guides PDF sont rendus via un viewer intégré (React-PDF ou PDF.js). Le bouton de téléchargement natif est masqué. Le fichier n'est jamais exposé à l'URL publique — il passe par une API route Next.js qui vérifie la session.
  • 🛡️
    Rate limiting & détection de session partagée Si un compte se connecte depuis plus de 2 appareils différents en 24h, il reçoit une alerte. Un compte partagé entre 10 esthéticiennes d'un salon est détecté et bloqué après 3 appareils simultanés.
